# API Key Setup - Architecture Clarification

## Key Hierarchy

There are **TWO separate keys** in this setup:

### 1. Anthropic API Key (One-time Setup)
- **Your personal Anthropic API key** (starts with `sk-ant-api03-...`)
- Registered once in SpendSteward
- Stored encrypted in SpendSteward database
- SpendSteward uses this to forward requests to Anthropic API
- **You provide it once**, then SpendSteward manages it

### 2. JWT Token (Per Session)
- **Authentication token** for SpendSteward API
- Generated by SpendSteward when you log in
- Used to authenticate with SpendSteward proxy endpoints
- **You get a new one each session** (expires after ~1 hour)

## Setup Flow

```
┌─────────────────────────────────────────────────────────┐
│ One-Time Setup                                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 1. You have Anthropic API key: sk-ant-api03-...        │
│                                                         │
│ 2. Register in SpendSteward:                            │
│    POST /api/v1/keys                                    │
│    {"provider": "anthropic", "api_key": "sk-ant..."}   │
│                                                         │
│ 3. SpendSteward encrypts and stores it                  │
│    ✅ Done - never need to do this again               │
│                                                         │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ Per-Session (Every Time You Run Agent)                  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 1. Get JWT token from SpendSteward:                     │
│    POST /auth/jwt/login                                 │
│    {"username": "test@example.com", ...}               │
│    → Returns JWT token: eyJhbGciOiJIUzI1NiIs...        │
│                                                         │
│ 2. Use JWT token to authenticate with proxy:           │
│    POST /api/v1/proxy/anthropic/messages               │
│    Header: Authorization: Bearer <JWT_TOKEN>           │
│                                                         │
│ 3. SpendSteward receives request:                       │
│    ├─ Verifies JWT token                               │
│    ├─ Retrieves Anthropic API key from DB              │
│    ├─ Forwards request to Anthropic with their key     │
│    ├─ Logs usage and costs                             │
│    └─ Returns response to you                          │
│                                                         │
│ ✅ Agent runs with costs tracked in SpendSteward       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## Why Two Keys?

**Anthropic API Key:**
- **What:** Your personal API key to Anthropic
- **Why:** SpendSteward needs it to forward requests
- **Security:** Encrypted at rest in database
- **How often:** Set once, reused forever
- **Who sees it:** You (initially), SpendSteward backend (encrypted)

**JWT Token:**
- **What:** SpendSteward session token
- **Why:** Authenticates your requests to SpendSteward
- **Security:** Temporary (expires ~1 hour), not stored
- **How often:** Get new one per session
- **Who sees it:** You, SpendSteward (in-memory)

## Implementation

### Already Done ✅

Your Anthropic API key (`sk-ant-api03-cwqoSeb...`) is already registered in SpendSteward.

### For Each Agent Run

Only provide the **JWT token** to the agent:

```bash
# Get JWT token
JWT_TOKEN=$(curl -s -X POST http://localhost:8000/auth/jwt/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d 'username=test@example.com&password=testpassword123' | \
  python3 -c "import sys, json; print(json.load(sys.stdin)['access_token'])")

# Run agent with JWT token (NOT Anthropic key)
./run_agent_with_proxy.sh $JWT_TOKEN
```

## Request Flow

```
Agent Code
    ↓
OpenAI SDK client.chat.completions.create()
    ↓ (uses GAMEDEV_AGENT_TOKEN=<JWT_TOKEN>)
POST http://localhost:8000/api/v1/proxy/anthropic/messages
  Header: Authorization: Bearer <JWT_TOKEN>
    ↓
SpendSteward Proxy receives JWT
  ├─ Decodes JWT → identifies user
  ├─ Loads Anthropic API key from database (encrypted)
  ├─ Performs rate limit check
  ├─ Checks cache
  ├─ Applies routing rules
    ↓
POST https://api.anthropic.com/v1/messages
  Header: x-api-key: sk-ant-api03-...
    ↓
Anthropic API processes request
    ↓
Response returned through proxy
  ├─ Log usage event
  ├─ Update cost tracking
  ├─ Cache response (if deterministic)
    ↓
Agent receives response
```

## Key Points

✅ **Anthropic key is encrypted** - stored securely in SpendSteward
✅ **JWT token is temporary** - get a new one each session
✅ **Agent only uses JWT token** - not the Anthropic key directly
✅ **SpendSteward handles Anthropic auth** - transparently forwards requests
✅ **Cost tracking is automatic** - happens inside SpendSteward proxy

## Common Mistakes to Avoid

❌ DON'T pass Anthropic key as GAMEDEV_AGENT_TOKEN
   ✅ DO pass JWT token as GAMEDEV_AGENT_TOKEN

❌ DON'T try to register Anthropic key each time
   ✅ DO register it once, then just use JWT tokens

❌ DON'T hardcode Anthropic key in scripts
   ✅ DO retrieve JWT token dynamically each session

❌ DON'T expose Anthropic key in environment
   ✅ DO store it encrypted in SpendSteward

## Summary

| When | Key | Usage |
|------|-----|-------|
| **One-time setup** | Anthropic API key | `POST /api/v1/keys` (register once) |
| **Each agent run** | JWT token | `./run_agent_with_proxy.sh <JWT>` |
| **Agent requests** | JWT token | Authorization header to proxy |
| **Proxy forwards** | Anthropic API key | x-api-key header to Anthropic |

The Anthropic key is managed by SpendSteward. You only interact with JWT tokens.
